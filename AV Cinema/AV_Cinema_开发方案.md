## 1. 项目概述

### 1.1 现状

JavTag 基于 Adobe AIR（Flash）技术构建，Adobe 已于 2019 年停止支持 AIR/Flex，技术严重过时。程序主体为编译后的 SWF（main.dll），无法直接修改源码，只能通过外部 DLL 扩展有限功能。

### 1.2 目标

使用现代桌面技术栈重写为"AV Cinema"，实现：

- ✅ 100% 保留现有功能
- ✅ 分类体系优化（合并/重命名/删除）
- ✅ 可维护、可扩展的代码架构
- ✅ 更美观的现代化 UI
- ✅ 跨平台支持（Windows/Mac）

### 1.3 核心数据规模

| 数据类型 | 数量 | 说明 |
|---------|------|------|
| 影片 | 641 部 | 含番号、女优、标签、截图路径、播放路径 |
| 女优 | 2,260 人 | 含出生日期、身高三围、罩杯、评分、头像 |
| 收藏 | 201 条 | 收藏的影片和女优 |
| 网站导航 | 8 个 | 外部资源网站链接 |
| 标签分类 | 1 套 | 7 大分类体系，含 100+ 细分类别 |
| 截图资源 | 数千张 | 存储在 resources/actress 目录 |

## 3. 分类体系调整方案

### 3.1 变更总览

```
调整前                     调整后
────────                   ────────
有码 ──→ 改名 → AV          AV（原：有码 + 无码 + 自拍）
无码 ──→ 删除，归入 AV      动漫（原：其他）
自拍 ──→ 删除（无数据）     欧美（不变）
其他 ──→ 改名 → 动漫       国产（不变）
欧美 ──→ 不变
国产 ──→ 不变
```

### 3.2 分类维度调整

**主分类（一级）**
- AV（原"有码"——日系成人影片主体）
- 动漫（原"其他"——动漫/动画类）
- 欧美（不变）
- 国产（不变）

**删除筛选维度：类型**
- 移除"类型"筛选行（原：中字/流出/含磁力）
- 对应的数据库字段 pj/ dt/ dm 保留在数据库中不做删除（兼容旧数据）
- 在新 UI 中不再展示这组筛选按钮

### 3.3 数据库迁移规则

**movies 表：**

```sql
UPDATE movies SET fl = 'AV'   WHERE fl = '有码';
UPDATE movies SET fl = 'AV'   WHERE fl = '无码';
UPDATE movies SET fl = '动漫' WHERE fl = '其他';
-- 国产/欧美 不变
```

**femas 表：**

```sql
-- 纯"有码"改为"AV"
UPDATE femas SET fl = 'AV' WHERE fl = '有码';

-- "有码，无码"去掉无码，改为"AV"  
UPDATE femas SET fl = 'AV' WHERE fl = '有码，无码';

-- 纯"无码"改为"AV"
UPDATE femas SET fl = 'AV' WHERE fl = '无码';

-- "无码，欧美"去掉无码，改为"欧美"
UPDATE femas SET fl = '欧美' WHERE fl = '无码，欧美';

-- "有码，欧美"改为"AV，欧美"
UPDATE femas SET fl = 'AV，欧美' WHERE fl = '有码，欧美';
```

**website 表：** 不变

**tagclas 表：** 不变

## 4. 技术选型方案

### 4.1 推荐方案一：Electron + Vue 3（推荐）

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| 桌面框架 | Electron 28+ | 生态最成熟，开发者最多，文档最全 |
| 前端框架 | Vue 3 + Composition API | 学习曲线低，中文社区大，适合快速开发 |
| UI 组件库 | Element Plus | 中文友好，组件丰富，开箱即用 |
| 构建工具 | Vite | 极快的 HMR 开发体验 |
| 数据库 | better-sqlite3 | 原生 SQLite 驱动，性能好 |
| 状态管理 | Pinia | Vue 3 官方推荐，简洁 |
| 视频播放 | FFmpeg + Electron 内置播放 | 复用现有 ffmpeg.dll |
| 打包 | electron-builder | 支持 Windows/Mac 打包 |
| CSS 框架 | Tailwind CSS | 灵活定制暗黑主题 |

**优点：** 生态最成熟，网上资源最多，遇到问题容易找到解决方案
**缺点：** 打包体积较大（约 80-120MB）

### 4.2 推荐方案二：Tauri + Vue 3

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| 桌面框架 | Tauri 2.0 | 打包体积小（约 5MB），性能好 |
| 前端框架 | Vue 3 + Composition API | 同上 |
| UI 组件库 | Element Plus | 同上 |
| 构建工具 | Vite | 同上 |
| 数据库 | rusqlite (Rust 端) | Tauri 内置 |
| 状态管理 | Pinia | 同上 |
| 视频播放 | FFmpeg 命令行调用 | Rust 调用系统 FFmpeg |
| 打包 | Tauri CLI | 原生支持多平台 |

**优点：** 打包体积极小，性能更好，安全性更高
**缺点：** Rust 语言相对小众，定制后端功能需要 Rust 知识

### 4.3 技术选型建议

> **建议选择方案一（Electron + Vue 3）**，原因如下：
>
> 1. 你提到"维护性差"是痛点，Electron 生态开发者最多，找人维护成本最低
> 2. Vue 3 中文社区非常活跃，遇到问题可以轻松搜索到答案
> 3. Element Plus 组件库中文文档齐全，上手快
> 4. 后续想加功能（在线刮削、自动更新等），Electron 插件生态更丰富

## 6. 数据库迁移方案

### 6.1 策略：直接读取旧数据库

建议 AV Cinema 直接读取并修改旧的 `data.dll`（SQLite 数据库），无需导出/导入数据。

**迁移步骤：**

1. **备份原始数据库**（防止操作失误）
   ```
   copy data.dll data.dll.backup
   ```

2. **执行数据迁移 SQL**（在 AV Cinema 首次启动时自动执行或手动执行）

3. **记录迁移版本**（在数据库中建一个 `schema_version` 表，记录已执行的迁移，方便后续升级）

### 6.2 SQL 迁移脚本

```sql
-- ============================================
-- AV Cinema 分类迁移脚本 v1.0
-- ============================================

-- Step 1: movies 表
UPDATE movies SET fl = 'AV'   WHERE fl = '有码';
UPDATE movies SET fl = 'AV'   WHERE fl = '无码';
UPDATE movies SET fl = '动漫' WHERE fl = '其他';
-- 国产、欧美 保持不变

-- Step 2: femas 表
UPDATE femas SET fl = 'AV'      WHERE fl = '有码';
UPDATE femas SET fl = 'AV'      WHERE fl = '有码，无码';
UPDATE femas SET fl = 'AV'      WHERE fl = '无码';
UPDATE femas SET fl = '欧美'    WHERE fl = '无码，欧美';
UPDATE femas SET fl = 'AV，欧美' WHERE fl = '有码，欧美';

-- Step 3: 记录迁移版本
CREATE TABLE IF NOT EXISTS _migration (
  version INTEGER PRIMARY KEY,
  desc TEXT,
  applied_at TEXT
);
INSERT INTO _migration (version, desc, applied_at)
VALUES (1, '分类调整：有码→AV，无码→AV，其他→动漫，删除类型筛选项', datetime('now'));
```

### 6.3 文件资源处理

女优头像路径中原包含 `[appUrl]` 占位符，在新软件中需要处理：

```
原路径: [appUrl]\META-INF\resources\actress\33.jpgs
新路径: ./resources/actress/33.jpgs  （相对于软件所在目录）
```

处理方式：
- 启动时自动检测 JavTag 原目录位置
- 或允许用户在设置中指定截图/资源的根目录
